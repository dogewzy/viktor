# AGENTS.md

本文件记录 AI Agent 在本项目中工作时需要牢记的公开版项目事实与默认行为约定。
不要在此文件中写入生产集群、数据库、跳板机、API key、token、个人信息或公司内部域名。

## 1. 部署拓扑

Viktor 可以部署为 Kubernetes Pod，也可以本地运行。开源仓库只保留通用部署约定：

| 项 | 值 |
| --- | --- |
| Kubernetes context | 通过本地 kubeconfig 或 `K8S_API_SERVER / K8S_TOKEN / K8S_CA_DATA` 注入 |
| Namespace | 通过 `K8S_NAMESPACE` 注入 |
| Pod 前缀 | 通过部署清单决定 |

## 2. 网络可达性

- 若 Viktor 与业务服务部署在同一集群或同一 VPC，注册的数据库、Redis、内部 HTTP 服务通常可直连。
- 本地不一定能访问内网资源；需要跳板机时，应在对应 Connector 中显式配置 `ssh_tunnel`。
- `.env` 仅用于本地开发，且必须保持未跟踪状态。
- 生产环境所有敏感配置通过 Secret/ConfigMap 或等价机制注入，不写入仓库。

## 3. Agent 行为约定

1. 默认直连；跳板机仅在数据库连接器显式声明时启用。
2. 新增外部连接时，敏感凭证只能来自环境变量或密钥系统。
3. Trace 复盘优先沉淀字段映射、统计口径、workflow 和反模式，不沉淀一次性数据值。

## 4. Git / 提交流程

- 公开仓库分支不得包含真实密钥、数据库连接串、内网域名、个人手机号、生产集群名称或私有项目名。
- 提交前运行敏感词扫描，并确认 `.env` 等本地配置未被跟踪。
